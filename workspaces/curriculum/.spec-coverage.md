None. All 80 exercises exist in all 4 formats. No exercises are missing.

### Major (should fix)

1. **M7 spec inaccuracy (7.1, 7.4, 7.6)**: Spec claims `TrainingPipeline` is used, but these exercises deliberately implement training from scratch. The spec should be updated to say "Manual implementation" or "ModelVisualizer" to match reality. This is a spec documentation error, not a code error -- the pedagogical design is correct.

### Significant (monitor)

3. **M3 3.5 ConditionalNode**: Spec lists it but implementation does not use it. The exercise already covers WorkflowBuilder + PythonCodeNode + DataFlow. Adding ConditionalNode would strengthen the workflow orchestration lesson.

4. **M6 6.5 CostTracker**: Spec lists it but implementation does not use it. Adding cost tracking to the governance exercise would be valuable.

5. **M4 4.4 EnsembleEngine methods**: Only `.blend()` is demonstrated. The spec mentions `stack()`, `bag()`, `boost()` which would give students broader ensemble coverage.

### Minor (accept with documentation)

6. **M1 1.1 dataset change**: Weather vs HDB. Pedagogically sound (simpler entry point). Update spec.
7. **M6 6.4 OnnxBridge**: Omitted from merging exercise. ONNX export is covered in M7/M8.
8. **M8 8.7 OnnxBridge**: Deferred to capstone (8.8). Not a real gap.
9. **M10 10.6 Address**: GovernanceEngine methods used instead. Minor API surface difference.
